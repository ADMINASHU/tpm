Key Recommendations for TPM Optimization

1. Database: Zero-Latency Lookups

Factory floors rely on instant scans. As your data grows, standard queries will slow down.

Implement Compound Indexing: Always index fields frequently used for filtering together (e.g., factory + createdAt). This prevents MongoDB from scanning every record in the collection.

Lean Queries: Use .lean() in all Mongoose GET requests. This skips the creation of complex Mongoose objects and returns plain JSON, reducing server memory usage by ~30%.

Data Projection: Only fetch the fields you actually need. Instead of find(), use find().select('name sku quantity').

2. API: Payload & Traffic Control

Large data sets should never be sent to the client in a single request.

Server-Side Pagination: Implement mandatory page and limit parameters for high-volume endpoints like Transactions and Assembly Logs.

Parallel Fetching: Use Promise.all() to fetch data and metadata (like total counts) simultaneously to reduce response wait times.

3. Frontend: "Light" UI Experience

A browser can struggle to render thousands of table rows, causing lag.

List Virtualization: For tables with hundreds of entries (e.g., Inventory Logs), use "Virtual Lists." This ensures only the rows currently visible on the screen are rendered in the DOM.

Optimistic Updates: Use a state management library like TanStack Query to show "success" immediately when a worker logs an entry, syncing with the database in the background.

Debounced Inputs: Optimize barcode scanner inputs so the application doesn't try to validate or search for every single character typed in rapid succession.

4. Heavy Processing: Offloading

Don't let complex calculations freeze your user interface.

Database Aggregations: Move heavy calculations (like Aging Reports or Costing) from JavaScript to MongoDB's Aggregation Framework ($group, $sum, $match).

Background Jobs: For tasks like PDF generation or bulk data imports, use background processing so the user can continue working while the task runs.
